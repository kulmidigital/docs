Create a hosted checkout session for popup, inline, or redirect payment flows.

**Method:** `POST`  
**URL:** `https://app.kulmipay.com/api/v1/checkout/`

For sandbox, use:

```text
https://sandbox.kulmipay.com/api/v1/checkout/
```

## Authentication

Pass your publishable key in the request body:

```json
{
  "public_key": "ISPubKey_live_xxxxxxxxxxxxxxxx"
}
```

Or pass it in the HTTP header:

```http
X-KULMI-PUBLIC-API-KEY: ISPubKey_live_xxxxxxxxxxxxxxxx
```

Do not use your secret key with this endpoint.

## Request

<ParamField body="public_key" type="string">
  Your publishable API key. Required unless you pass `X-KULMI-PUBLIC-API-KEY`.
</ParamField>

<ParamField body="amount" type="number" required>
  Amount to collect. For UGX and TZS, the amount must be between 500 and 5,000,000.
</ParamField>

<ParamField body="currency" type="string">
  Currency for the checkout session. Defaults to your account payment settings when omitted.
</ParamField>

<ParamField body="email" type="string">
  Customer email address.
</ParamField>

<ParamField body="phone_number" type="string">
  Customer phone number in international format, for example `254712345678`.
</ParamField>

<ParamField body="first_name" type="string">
  Customer first name.
</ParamField>

<ParamField body="last_name" type="string">
  Customer last name.
</ParamField>

<ParamField body="api_ref" type="string">
  Your order or transaction reference.
</ParamField>

<ParamField body="redirect_url" type="string">
  URL where the customer should return after payment.
</ParamField>

<ParamField body="callback_url" type="string">
  Compatibility field sent by the JavaScript SDK. Use `redirect_url` for customer redirects.
</ParamField>

<ParamField body="host" type="string">
  Merchant website origin. The JavaScript SDK sends this automatically from the browser location.
</ParamField>

<ParamField body="method" type="string">
  Restrict checkout to a method when supported by your account settings.
</ParamField>

<ParamField body="mobile_tarrif" type="string" default="BUSINESS-PAYS">
  Who pays mobile money fees: `BUSINESS-PAYS` or `CUSTOMER-PAYS`.
</ParamField>

<ParamField body="bank_tarrif" type="string" default="BUSINESS-PAYS">
  Who pays bank transfer fees: `BUSINESS-PAYS` or `CUSTOMER-PAYS`.
</ParamField>

<ParamField body="wallet_id" type="string">
  Wallet to fund directly, when wallet funding is enabled for your account.
</ParamField>

<ParamField body="unique_api_ref" type="boolean" default="false">
  When `true`, KulmiPay returns an existing checkout with the same `api_ref` instead of creating a duplicate.
</ParamField>

## Response

<ResponseField name="id" type="string">
  Checkout session UUID.
</ResponseField>

<ResponseField name="url" type="string">
  Hosted checkout URL. Redirect the customer here when you are not using the browser SDK.
</ResponseField>

<ResponseField name="signature" type="string">
  Signed token for the checkout session. The browser SDK passes this to the hosted checkout UI.
</ResponseField>

<ResponseField name="tracking_id" type="string">
  Invoice tracking ID after a payment invoice is associated with the checkout.
</ResponseField>

<ResponseField name="paid" type="boolean">
  Whether the checkout has been paid.
</ResponseField>

<ResponseField name="methods" type="array">
  Payment methods returned from your payment settings.
</ResponseField>

<ResponseField name="layout" type="string">
  Checkout layout returned from your payment settings.
</ResponseField>

<ResponseField name="styles" type="object">
  Checkout theme styles returned from your payment settings.
</ResponseField>

<ResponseField name="merchant_name" type="string">
  Merchant descriptor shown to the customer.
</ResponseField>

<ResponseField name="merchant_origin" type="string">
  Merchant origin derived from your business website or request host.
</ResponseField>

## Code examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://app.kulmipay.com/api/v1/checkout/ \
  --header 'Content-Type: application/json' \
  --data '{
    "public_key": "ISPubKey_live_xxxxxxxxxxxxxxxx",
    "amount": 2500,
    "currency": "KES",
    "email": "jane.doe@example.com",
    "phone_number": "254712345678",
    "api_ref": "ORDER-1001",
    "redirect_url": "https://merchant.example/thank-you",
    "unique_api_ref": true
  }'
```

```javascript JavaScript
const response = await fetch("https://app.kulmipay.com/api/v1/checkout/", {
  method: "POST",
  headers: {
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    public_key: "ISPubKey_live_xxxxxxxxxxxxxxxx",
    amount: 2500,
    currency: "KES",
    email: "jane.doe@example.com",
    phone_number: "254712345678",
    api_ref: "ORDER-1001",
    redirect_url: "https://merchant.example/thank-you",
    unique_api_ref: true
  })
});

const checkout = await response.json();
console.log(checkout.url);
```

```python Python
import requests

response = requests.post(
    "https://app.kulmipay.com/api/v1/checkout/",
    json={
        "public_key": "ISPubKey_live_xxxxxxxxxxxxxxxx",
        "amount": 2500,
        "currency": "KES",
        "email": "jane.doe@example.com",
        "phone_number": "254712345678",
        "api_ref": "ORDER-1001",
        "redirect_url": "https://merchant.example/thank-you",
        "unique_api_ref": True,
    },
)

checkout = response.json()
print(checkout["url"])
```

## Checkout details

Use `POST /api/v1/checkout/details/` to fetch checkout details for an existing checkout session.

```json
{
  "checkout_id": "d9f3a821-5b2c-4e8a-a1d4-3c7f9e2b0a14",
  "signature": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

<Note>
  The `kulmipay` browser SDK handles checkout creation and checkout details for you. Use this API directly when you are building a custom redirect or server-rendered flow.
</Note>
