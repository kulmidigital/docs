Use this endpoint to retrieve the webhook configurations registered on your KulmiPay account. Each webhook endpoint subscribes to one or more event types. When a matching event occurs, KulmiPay sends an HTTP `POST` request to your endpoint containing the event payload.

Webhook creation and management are handled in the KulmiPay dashboard.

**Method:** `GET`  
**URL:** `https://app.kulmipay.com/api/v1/webhooks/`

Webhook endpoints must use HTTPS. Endpoints that cannot be reached or return non-2xx responses repeatedly are disabled after reaching the failure threshold.

## Code example

```bash cURL
curl --request GET \
  --url https://app.kulmipay.com/api/v1/webhooks/ \
  --header "Authorization: Bearer YOUR_SECRET_KEY"
```

## Webhook object

<ResponseField name="id" type="string">
  Unique identifier for this webhook configuration.
</ResponseField>

<ResponseField name="endpoint" type="string">
  The HTTPS URL where KulmiPay sends event payloads.
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Whether this webhook is currently active. KulmiPay sets this to `false` automatically when the endpoint reaches the failure threshold.
</ResponseField>

<ResponseField name="collection_event" type="boolean">
  When `true`, this webhook receives notifications whenever a collection invoice changes state (for example, when a payment completes or fails).
</ResponseField>

<ResponseField name="send_money_event" type="boolean">
  When `true`, this webhook receives Send Money payment file state changes.
</ResponseField>

<ResponseField name="reversal_event" type="boolean">
  When `true`, this webhook receives notifications when a chargeback or reversal is processed on your account.
</ResponseField>

<ResponseField name="wallet_transfer_event" type="boolean">
  When `true`, this webhook receives notifications when an intra-wallet transfer is completed.
</ResponseField>

<ResponseField name="failures_count" type="number">
  The number of consecutive delivery failures for this endpoint. KulmiPay warns you by email when this reaches 50% of the failure threshold, and disables the endpoint at 100%.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when this webhook was registered.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the last configuration update.
</ResponseField>

```json
  "results": [
    {

      "endpoint": "https://yourapp.example.com/webhooks/kulmipay",
      "is_active": true,

      "reversal_event": false,
      "wallet_transfer_event": false,
      "failures_count": 0,

      "updated_at": "2024-04-10T09:30:00Z"
    }

```

<Note>
  The `challenge` value used to verify incoming webhook payloads is not returned by this endpoint. Store it securely when you register the webhook. KulmiPay includes it as `challenge` in each webhook payload.
</Note>

## Verifying webhook payloads

Before processing an event, compare the payload's `challenge` field against the challenge value you configured in the dashboard.
