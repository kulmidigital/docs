---
title: "POST /api/v1/send-money/status"
sidebarTitle: "Check send money status"
description: "Retrieve the current status of a Send Money payment file and each of its individual transactions using the tracking_id returned at initiation."
---

Use this endpoint to check the state of a payment file and each of its transactions. You can poll this endpoint after initiating or approving a Send Money request to confirm whether individual transactions have completed, are still processing, or have failed.

You must have the `view_files` permission on your API token to call this endpoint.

## Request

**`POST https://app.kulmipay.com/api/v1/send-money/status/`**

<ParamField header="Authorization" type="string" required>
  Secret key. Format: `Bearer ISSecretKey_test_xxxxxxxxxxxxxxxx`.
</ParamField>

<ParamField body="tracking_id" type="string" required>
  The UUID returned as `tracking_id` in the initiation response. Use this to identify the file to query.
</ParamField>

## Response

A successful request returns HTTP `200` with the full file object and its transactions.

<ResponseField name="file_id" type="string">
  The payment file identifier.
</ResponseField>

<ResponseField name="tracking_id" type="string">
  The file's tracking UUID, matching what you submitted.
</ResponseField>

<ResponseField name="status" type="string">
  Human-readable status of the payment file. Common values:

  | Status | Meaning |
  |---|---|
  | `Preview and approve` | File created, awaiting approval |
  | `Confirming balance` | Kulmi Pay is checking the source wallet balance |
  | `Sending payment` | Transactions are being queued for processing |
  | `Processing payment` | Transactions are being processed by the provider |
  | `Completed` | All transactions finished, with or without failures |
  | `Cancelled` | File was cancelled before processing completed |
</ResponseField>

<ResponseField name="status_code" type="string">
  Machine-friendly Kulmi Pay status code for the payment file, such as `BP103`, `BP104`, `BP109`, `BP110`, `BC100`, or `BE111`.
</ResponseField>

<ResponseField name="total_amount" type="string">
  Total value of all transactions in the file.
</ResponseField>

<ResponseField name="transactions_count" type="number">
  Number of transactions in the file.
</ResponseField>

<ResponseField name="transactions" type="array">
  The individual transactions and their statuses.

  <Expandable title="Transaction object fields">
    <ResponseField name="transaction_id" type="string">
      Unique identifier for this transaction.
    </ResponseField>
    <ResponseField name="status" type="string">
      Human-readable transaction status, such as `Pending`, `Payment initiated`, `Processing results`, `Successful`, `Unsuccessful`, or `Cancelled`.
    </ResponseField>
    <ResponseField name="status_code" type="string">
      Machine-friendly Kulmi Pay transaction status code, such as `TP101`, `TP102`, `TP104`, `TS100`, `TF106`, or `TC108`.
    </ResponseField>
    <ResponseField name="status_description" type="string">
      Human-readable reason for the current status, especially useful for failed transactions.
    </ResponseField>
    <ResponseField name="account" type="string">
      Recipient phone number or account number.
    </ResponseField>
    <ResponseField name="amount" type="string">
      Transaction amount.
    </ResponseField>
    <ResponseField name="name" type="string">
      Recipient name.
    </ResponseField>
    <ResponseField name="bank_code" type="string">
      Bank code, for PesaLink transactions.
    </ResponseField>
    <ResponseField name="narrative" type="string">
      Payment description.
    </ResponseField>
    <ResponseField name="provider_reference" type="string">
      The reference number assigned by the payment provider (e.g. the M-Pesa transaction ID).
    </ResponseField>
    <ResponseField name="provider_account_name" type="string">
      The account name returned by the provider for the recipient, where available.
    </ResponseField>
    <ResponseField name="charge" type="string">
      Transaction fee charged for this individual transaction.
    </ResponseField>
    <ResponseField name="currency" type="string">
      Currency of the transaction.
    </ResponseField>
    <ResponseField name="idempotency_key" type="string">
      The idempotency key submitted with this transaction, if any.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp when the transaction was created.
    </ResponseField>
    <ResponseField name="updated_at" type="string">
      ISO 8601 timestamp of the last status update.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="actual_charges" type="string">
  Total charges applied to the file.
</ResponseField>

<ResponseField name="paid_amount" type="string">
  Total amount successfully paid out.
</ResponseField>

<ResponseField name="failed_amount" type="string">
  Total amount from failed transactions.
</ResponseField>

<ResponseField name="wallet" type="object">
  The wallet debited for this file.
</ResponseField>

<ResponseField name="charge_estimate" type="string">
  Estimated charges at time of initiation.
</ResponseField>

<ResponseField name="batch_reference" type="string">
  Your batch reference, as submitted.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp when the file was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the last file status update.
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl -X POST https://app.kulmipay.com/api/v1/send-money/status/ \
  -H "Authorization: Bearer ISSecretKey_test_xxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "tracking_id": "d9e4f5a6-1234-5678-abcd-ef0123456789"
  }'
```

```python Python
import requests

response = requests.post(
    "https://app.kulmipay.com/api/v1/send-money/status/",
    headers={
        "Authorization": "Bearer ISSecretKey_test_xxxxxxxxxxxxxxxx",
        "Content-Type": "application/json",
    },
    json={
        "tracking_id": "d9e4f5a6-1234-5678-abcd-ef0123456789",
    },
)
print(response.json())
```

</CodeGroup>
