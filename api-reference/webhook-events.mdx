Use this endpoint to inspect every event KulmiPay attempted to deliver to your webhook endpoints. Each record shows the topic, payload, delivery status, and, for failed deliveries, the reason for failure. Use this log to debug integration issues.

**Method:** `GET`  
**URL:** `https://app.kulmipay.com/api/v1/webhooks-events/`

Results are ordered from newest to oldest.

## Code example

```bash cURL
curl --request GET \
  --url https://app.kulmipay.com/api/v1/webhooks-events/ \
  --header "Authorization: Bearer YOUR_SECRET_KEY"
```

## Event object

<ResponseField name="id" type="string">
  Unique identifier for this event record.
</ResponseField>

<ResponseField name="webhook" type="object">
  The webhook configuration this event was sent to.

  <Expandable title="webhook properties">
    <ResponseField name="id" type="string">
      The webhook configuration ID.
    </ResponseField>
    <ResponseField name="endpoint" type="string">
      The URL KulmiPay attempted to deliver this event to.
    </ResponseField>
    <ResponseField name="is_active" type="boolean">
      Whether the webhook endpoint is still active.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topic" type="string">
  The event type. One of:
  - `collection_event` — a payment collection changed state
  - `send_money_event` — a Send Money payment file changed state
  - `reversal_event` — a chargeback or reversal was processed
  - `wallet_transfer_event` — an intra-wallet transfer completed
</ResponseField>

<ResponseField name="payload" type="string">
  The JSON payload that was (or will be) delivered to your endpoint. This is the complete event data, serialized as a JSON string.
</ResponseField>

<ResponseField name="status" type="string">
  Delivery status of this event. One of:
  - `PENDING` — queued, not yet attempted
  - `PROCESSING` — currently being delivered
  - `DELIVERED` — successfully delivered (your endpoint returned a 2xx response)
  - `FAILED` — all delivery attempts failed
</ResponseField>

<ResponseField name="failure_reason" type="string">
  The error message from the most recent failed delivery attempt. Present only when `status` is `FAILED`. Truncated to 300 characters.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when this event was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the last status update for this event.
</ResponseField>
