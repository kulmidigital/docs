---
title: "GET/POST /api/v1/subscriptions — Manage Subscriptions"
description: "Create and manage recurring billing subscriptions that automatically charge customers on a schedule defined by a billing plan you configure."
---

Use these endpoints to create, retrieve, update, and cancel subscriptions. A subscription links a customer to a billing plan and automatically charges the customer's saved card on the schedule defined by the plan. New subscriptions start in `PENDING` status and become `ACTIVE` after the customer completes card setup.

**Base URL:** `https://app.kulmipay.com/api/v1/subscriptions/subscriptions/`

## Subscription lifecycle

| Status | Description |
| ------ | ----------- |
| `PENDING` | Subscription created, awaiting customer card setup. |
| `ACTIVE` | Card setup complete; recurring billing is running. |
| `COMPLETE` | All billing cycles in the plan have been processed. |
| `CANCELED` | Subscription was cancelled via the unsubscribe endpoint. |
| `FAILED` | A billing attempt failed. The subscription retries on the next scheduled date. |

## Create a subscription

**`POST /api/v1/subscriptions/subscriptions/`**

Creates a new subscription for a customer against a billing plan. The response includes a `setup_url` — send your customer to this URL to complete card enrollment. The subscription becomes `ACTIVE` once the customer successfully sets up their payment method.

### Request parameters

<ParamField body="plan_id" type="string" required>
  The `plan_id` (alias ID) of the billing plan to subscribe to. Create plans using the [plans endpoint](/api-reference/plans).
</ParamField>

<ParamField body="customer_id" type="string" required>
  The `customer_id` (alias ID) of the customer being subscribed. Create customers using the [customers endpoint](/api-reference/customers).
</ParamField>

<ParamField body="reference" type="string">
  Your internal reference ID for this subscription. Optional. Alphanumeric and standard punctuation characters only, up to 45 characters.
</ParamField>

<ParamField body="redirect_url" type="string">
  A URL to redirect the customer to after they complete card setup. If not provided, the plan's `redirect_url` is used.
</ParamField>

<ParamField body="start_date" type="string">
  The date on which billing should start, in `YYYY-MM-DD` format. Defaults to today if not specified.
</ParamField>

```bash cURL
curl --request POST \
  --url https://app.kulmipay.com/api/v1/subscriptions/subscriptions/ \
  --header "Authorization: Bearer YOUR_SECRET_KEY" \
  --header "Content-Type: application/json" \
  --data '{
    "plan_id": "pln_abc123",
    "customer_id": "cus_xyz789",
    "reference": "sub_internal_001"
  }'
```

## Get subscription details

**`GET /api/v1/subscriptions/subscriptions/{subscription_id}/`**

Returns the full details of a subscription, including the linked customer and plan objects.

## Update a subscription

**`PUT /api/v1/subscriptions/subscriptions/{subscription_id}/`**

Updates the plan or next billing date for an active subscription.

<ParamField body="plan_id" type="string" required>
  The `plan_id` of the new plan to switch this subscription to.
</ParamField>

<ParamField body="start_date" type="string">
  Override the next billing date, in `YYYY-MM-DD` format. Must be today or a future date.
</ParamField>

## Cancel a subscription

**`POST /api/v1/subscriptions/subscriptions/{subscription_id}/unsubscribe/`**

Cancels the subscription immediately. No further charges will be made. The subscription status changes to `CANCELED`.

```bash cURL
curl --request POST \
  --url https://app.kulmipay.com/api/v1/subscriptions/subscriptions/sub_abc123/unsubscribe/ \
  --header "Authorization: Bearer YOUR_SECRET_KEY"
```

## Get subscription payment history

**`GET /api/v1/subscriptions/subscriptions/{subscription_id}/transactions/`**

Returns a list of all payment records for this subscription, including the linked invoice for each billing cycle.

## Subscription object

<ResponseField name="subscription_id" type="string">
  The subscription alias ID. Use this value in all subsequent calls to reference this subscription.
</ResponseField>

<ResponseField name="status" type="string">
  Current subscription status: `PENDING`, `ACTIVE`, `COMPLETE`, `CANCELED`, or `FAILED`.
</ResponseField>

<ResponseField name="start_date" type="string">
  The date billing started, in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="next_date" type="string">
  The date of the next scheduled billing attempt. `null` if the subscription is complete or canceled.
</ResponseField>

<ResponseField name="completed_cycles" type="number">
  The number of billing cycles that have been successfully processed.
</ResponseField>

<ResponseField name="fail_reason" type="string">
  The reason for the most recent failed billing attempt. Present only when `status` is `FAILED`.
</ResponseField>

<ResponseField name="setup_url" type="string">
  The URL to send the customer to for card enrollment. Required for the subscription to become active.
</ResponseField>

<ResponseField name="plan" type="object">
  The billing plan linked to this subscription. See [plans](/api-reference/plans) for the full object structure.
</ResponseField>

<ResponseField name="customer" type="object">
  The customer linked to this subscription. See [customers](/api-reference/customers) for the full object structure.
</ResponseField>

<ResponseField name="payment_method" type="string">
  The payment method used for recurring charges. Currently `CARD-PAYMENT`.
</ResponseField>

<ResponseField name="card_mask" type="string">
  The masked card number used for billing, for example `****4242`. Present after card setup completes.
</ResponseField>

<ResponseField name="card_type" type="string">
  The card network, for example `VISA` or `MASTERCARD`. Present after card setup completes.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the subscription was created.
</ResponseField>
